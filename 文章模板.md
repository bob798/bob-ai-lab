# 文章模板

这个模板适合 BobAILab 当前的大多数文章。目标不是把所有文章写成一个样子，而是固定住最稳定的骨架，降低你每次开写的启动成本。

## 通用模板

```md
# 标题

适合谁看：

- 这篇最适合的第一类读者
- 这篇最适合的第二类读者
- 这篇最适合的第三类读者

这篇文章想讲清楚的核心是：

**一句话核心判断**

## 一、先讲问题或误区

- 大多数人怎么理解这件事
- 这个理解为什么不够好
- 这篇文章为什么值得继续读

## 二、给出你的核心判断

- 直接说结论
- 解释为什么这样判断
- 说清楚这和常见看法有什么不同

## 三、展开结构或方法

- 如果是方法文：讲框架、步骤、判断标准
- 如果是观点文：讲论证链和边界
- 如果是案例文：讲事情经过、误判点、转折点

## 四、给例子或真实场景

- 至少一个具体案例
- 最好有“错误示例 vs 更好示例”
- 如果是案例文，这部分就是主体

## 五、收束成可带走的东西

- 一句压缩后的判断
- 一个最小可执行动作
- 或一张检查清单 / 判断表

## 继续阅读

- 相关上一篇
- 相关下一篇
```

## 类型微调

### 方法文

重点补：

- 这个方法解决什么问题
- 什么时候适用
- 什么时候不适用
- 读者下一步怎么做

### 案例复盘文

重点补：

- 事情怎么发生
- 为什么会误判
- 后来怎么纠偏
- 提炼出什么可复用方法

### 观点文

重点补：

- 你反对的常见看法是什么
- 你的新判断是什么
- 论证链是否足够完整

### 指南文

重点补：

- 适用对象
- 步骤
- 判断标准
- 常见错误
- 最小行动清单

## 当前最适合 BobAILab 的固定结构

如果你想进一步固定下来，我建议优先统一这四个块：

1. `适合谁看`
2. `一句话核心判断`
3. `最小可执行动作 / 检查清单`
4. `继续阅读`

这四个块最适合你的写作风格，因为它们既能保留你的经验密度，也能明显增强读者视角。
